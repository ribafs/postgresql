Tutorial - Introduction to Indexes 
To create an index, the syntax is similar to creating a table:

indextest=# h CREATE INDEX
Command:     CREATE INDEX
Description: define a new index
Syntax:
CREATE [ UNIQUE ] INDEX index_name ON table
    [ USING acc_method ] ( column [ ops_name ] [, ...] )
    [ WHERE predicate ]
CREATE [ UNIQUE ] INDEX index_name ON table
    [ USING acc_method ] ( func_name( column [, ... ]) [ ops_name ] )
    [ WHERE predicate ]

Let's create an index on the id column:

pgsql@elrond ~$ time echo "CREATE INDEX id_index ON indextest(id)" | psql -d indextest
CREATE INDEX

real    2m31.495s
user    0m0.030s
sys     0m0.030s

It took us around two and one half minutes to create the index.

When we create an index, it must be given a name. This is mainly so we can see at a glance with the di command in psql what
indexes we have on what particular tables:

indextest=# di
               List of relations
 Schema |   Name   | Type  | Owner |   Table
--------+----------+-------+-------+-----------
 public | id_index | index | pgsql | indextest
(1 row)

To display what columns the index was created on, use the d <tablename> command:

indextest=# d indextest
     Table "public.indextest"
 Column |     Type     | Modifiers
--------+--------------+-----------
 id     | integer      |
 name   | text         |
 number | integer      |
 letter | character(1) |
Indexes: id_index btree (id)

As we can see, the index is called id_index and it was created on the id column.

Let's check the speed again:

pgsql@elrond ~$ time echo "SELECT * FROM indextest WHERE id='50000'" | psql -d indextest

  id   |   name   |  number  | letter
-------+----------+----------+--------
 50000 | WvMw5oZN | 53641070 | d
(1 row)

real    0m0.344s
user    0m0.030s
sys     0m0.020s

It has gone from 30 seconds to around a third of a second! That's a great performance improvement just by creating a single
index.


Removing Indexes

Removing indexes can speed up updating or adding data to tables. Since an index can't be altered (you can't add another column to
or remove a column from an index), to change an index it has to be removed then re-added. Dropping an index is quite easy:

indextest=# DROP INDEX id_index;
